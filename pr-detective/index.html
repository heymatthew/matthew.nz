TIL that you can find the PR which merged a commit using git log. It’s now a handy alias.

TLDR:

git config –global alias.trace ‘!git log $1..origin/head –ancestry-path –merges –reverse’ Which lets you do stuff like: Î» git trace f657f2af17

Why?

Git log usually shows you a history of what lead up to a change, but if you use the –ancestry-path flag you get it the other way round– what came after this thing. You can then filter further to bring up only merges with –merges. But it’s also most recent first which can be reversed with –reverse. Generally speaking I’m looking oldest commit first because these commits are usually way in the past. The result is a list of merges that happened after a commit. Github merge PR# can be found in the message.
